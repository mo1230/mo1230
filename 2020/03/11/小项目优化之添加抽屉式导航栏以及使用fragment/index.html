    小项目优化之添加抽屉式导航栏以及使用fragment

    移动端开发/ Android 历史上的今天 抽屉式导航栏
    创建时间:2020-03-11 17:48
    字数:1.1k 阅读:
    1. 抽屉式导航栏
    2. Fragment的管理

    这次是对前段时间的“历史上的今天”项目的部分优化，添加了抽屉式导航栏以及使用了Fragment（碎片）。

    抽屉式导航栏

    抽屉式导航栏就是那种从左侧滑出的导航栏。需要使用DrawerLayout以及NavigationView

    1. 添加依赖(NavigationView的依赖)
        implementation 'com.google.android.material:material:1.0.0'

    1. 编写布局文件

      • 页面布局

              <?xml version="1.0" encoding="utf-8"?>
      <!--这个布局文件就是带有抽屉式导航栏的页面的布局-->
      <!--1. 总体上使用DrawerLayout布局方式，这样整个页面就形成了一个抽屉-->
      <!--2. DrawerLayout布局分为两部分：
          第一部分容器是页面内容，
          第二部分是NavigationView，也就是侧边导航栏部分-->

      <androidx.drawerlayout.widget.DrawerLayout xmlns:android="http://schemas.android.com/apk/res/android"
          xmlns:app="http://schemas.android.com/apk/res-auto"
          xmlns:tools="http://schemas.android.com/tools"
          android:layout_width="match_parent"
          android:layout_height="match_parent"
          tools:context=".MainActivity"
          android:id="@+id/drawer">

      <!--    这个LinearLayout作为一个容器，是页面内容部分-->
          <LinearLayout
              android:layout_width="match_parent"
              android:layout_height="wrap_content"
              android:orientation="vertical">
              <androidx.appcompat.widget.Toolbar
                  android:id="@+id/toolbar"
                  android:layout_width="match_parent"
                  android:layout_height="wrap_content"
                  app:title="@string/lishi"

                  android:background="@color/colorAccent"
                  />
      <!--        这个RelativeLayout是用来添加、替换Fragment用的-->
              <RelativeLayout
                  android:id="@+id/fragments"
                  android:layout_width="match_parent"
                  android:layout_height="match_parent">

              </RelativeLayout>
          </LinearLayout>

      <!--    Navigation  侧边导航栏部分-->
      <!--    有几个属性必须要设置-->
      <!--    1. headerLayout  侧边导航栏的头部布局-->
      <!--    2. menu  侧边导航栏的菜单-->
      <!--    3. layout_gravity需要设置为start  设置的是侧边导航栏的滑出方向-->
          <com.google.android.material.navigation.NavigationView
              android:layout_width="match_parent"
              android:layout_height="match_parent"
              android:orientation="vertical"
              app:headerLayout="@layout/header"
              app:menu="@menu/navigation_menu"
              android:layout_gravity="start"
              android:id="@+id/nav">

          </com.google.android.material.navigation.NavigationView>
      </androidx.drawerlayout.widget.DrawerLayout>

      <!--也就是说DrawerLayout的组成为-->
      <!-- DrawerLayout：页面内容 + NavigationView,
          而NavigationView分为headerLayout + menu(headerLayout和menu作为属性设置)-->

      • 侧边导航栏头布局

              <?xml version="1.0" encoding="utf-8"?>
      <RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
          xmlns:app="http://schemas.android.com/apk/res-auto"
          xmlns:tools="http://schemas.android.com/tools"
          android:layout_width="match_parent" android:layout_height="match_parent">

          <ImageButton
              android:id="@+id/image"
              android:layout_width="100dp"
              android:layout_height="100dp"
              android:src="@drawable/me"
              android:padding="10dp"
              android:scaleType="centerCrop"
              android:layout_centerHorizontal="true"
              android:layout_marginTop="15dp"/>

      </RelativeLayout>

      • 侧边导航栏菜单文件

              <?xml version="1.0" encoding="utf-8"?>
      <menu xmlns:android="http://schemas.android.com/apk/res/android">
          <item android:id="@+id/hoem"
              android:title="主页"
              android:checkable="true"
              />

          <item android:id="@+id/setting"
              android:title="设置"
              />

          <item android:title="退出"
              android:id="@+id/quit"
              />
      </menu>

    1. 给导航栏添加事件
      (注意：这里有些代码被省略了，例如类的属性的声明)

       // 抽屉式导航栏
     this.navigationView = findViewById(R.id.nav);
     this.navigationView.setCheckedItem(R.id.hoem);
     this.drawerLayout = findViewById(R.id.drawer);
     // 侧边导航栏的菜单选择事件
     this.navigationView.setNavigationItemSelectedListener(new NavigationView.OnNavigationItemSelectedListener() {
         @Override
         public boolean onNavigationItemSelected(@NonNull MenuItem menuItem) {
             // 定义了对Fragment进行管理的对象
             FragmentManager fragmentManager = getSupportFragmentManager();
             FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
             switch (menuItem.getItemId()){
                 case R.id.hoem:
                     // 替换fragment
                     fragmentTransaction.replace(R.id.fragments, new HomeContent());
                     fragmentTransaction.addToBackStack(null);
                     fragmentTransaction.commit();
                     // 关闭抽屉
                     MainActivity.this.drawerLayout.closeDrawer(GravityCompat.START);
                     break;
                 case R.id.setting:

                     fragmentTransaction.replace(R.id.fragments, new Setting());
                     fragmentTransaction.addToBackStack(null);
                     fragmentTransaction.commit();

                     menuItem.setCheckable(true);

                     MainActivity.this.drawerLayout.closeDrawer(GravityCompat.START);
                     break;
                 case R.id.quit:
                     // 退出
                     MainActivity.this.finish();
                     break;
             }
             return true;
         }
     });

      以上就是有关抽屉式导航栏的设置

    Fragment的管理

    fragment即为碎片，它也有布局文件以及java文件，可以在一个activity下加入、删除、替换多个fragment。并且可以解决平板电脑上的布局问题

    1. 新建fragment以及布局文件
      使用Android Studio可以同时建立
    2. 在fragment的java文件中实现业务逻辑，在布局文件中设置布局页面
    3. 在activity中管理fragment
           // 添加碎片
         FragmentManager fragmentManager = getSupportFragmentManager();
         FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
         // R.id.fragments为activity的布局文件中的一个容器（例如LinearLayout），HomeContent是前面新建的碎片
         fragmentTransaction.add(R.id.fragments, new HomeContent());
         fragmentTransaction.commit();
